# Alerts

## Purpose

Alerts make important risk, staleness, and anomaly conditions visible without requiring the non-technical user to inspect diagnostic pages.

## Core Capabilities

- show forecast risk warnings
- show missing income or obligation warnings
- show sync and auth health issues
- direct admin-level issues to the technical operator

## Key Requirements

- `FR-21`
- `FR-30`
- `FR-31`

## UX Goal

Warnings should be understandable first, diagnostic second.
