# Auditability

## Purpose

Auditability ensures important actions and calculated outcomes can be traced after the fact, especially when the admin user needs to understand why the app behaved a certain way.

## Core Capabilities

- record auth events
- record configuration and override events
- preserve explanation trail for forecast-affecting actions
- keep source lineage for transactions and derived outputs

## Key Requirements

- `FR-9`
- `FR-28`
- `FR-31`
- `FR-32`

## Scope Note

This is household-grade auditability for trust and diagnosis, not enterprise compliance management.
