# User Story: Use Budgeting As Spending Guidance

## Story

As the technical household operator,
I want to define discretionary budget targets,
So that safe-to-spend guidance can reflect category-level spending pressure without hiding the raw forecast.

## Source Inputs

- feature dossier: `features/08_budgeting.md`
- FR references: `FR-23`, `FR-24`, `FR-25`

## Acceptance Notes

- budgets can be created and edited
- current discretionary spend versus plan is visible
- budgeting remains a guidance layer, not a black-box override

## Required Tests

- budget target maintenance
- discretionary comparison logic
- guidance visibility behavior
