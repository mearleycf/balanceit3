# Workflow Outputs

This directory turns the canonical BalanceIt3 docs into execution-ready artifacts.

## Rules

- start from the numbered canonical docs and the relevant feature dossier
- cite source files in every downstream artifact
- keep templates generic and examples concrete
- distinguish `required now` from `future hardening`
- preserve traceability back to FR identifiers where possible

## Workflow Order

1. epics
2. user stories
3. acceptance criteria
4. implementation plans
5. GitHub issue drafts
6. Linear issue drafts
7. sprint packets
8. testing packs
