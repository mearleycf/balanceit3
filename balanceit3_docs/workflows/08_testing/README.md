# Testing Workflows

This directory groups test planning artifacts by validation intent rather than creating a top-level folder for every possible test type.

## Testing Categories Covered

- unit
- integration
- end-to-end
- component and functional
- API and contract
- acceptance and regression
- smoke
- security and authentication
- performance and scalability
- cross-device and responsive
- database
- monitoring and alert validation

## Required Pattern

Each testing artifact should:

- reference the feature dossier and FR identifiers it covers
- state what must be tested now
- state what can wait as future hardening
- identify the main risks if coverage is missing
